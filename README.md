Лабораторная работа №2. Динамическое распределение памяти. Выполнил студент группы ИДБ-24-03 - Маслов И.С.

Условие задачи: Заменить все пробелы в строке на символ подчеркивания («_»), если строка содержит хотя бы одну цифру. Иначе — заменить их на «-».

Описание проекта:
Проект реализует программы для обработки строки символов с использованием механизмов динамического распределения памяти в C: malloc, calloc, realloc и free.

Проект включает три варианта реализации ввода и обработки строки:
1. Вариант 1 — ввод строки с использованием malloc + free(array.cpp)
* Память выделяется при каждом вводе символа.
* Для первого символа вызывается malloc.
* Для каждого следующего:
  1. Выделяется новый блок размером (старый размер + 1 символ + '\0').
  2. Ранее введённые данные вручную копируются в новый буфер.
  3. Старый буфер освобождается через free.
* После завершения ввода строка анализируется:
 * Определяется, содержит ли она цифры;
 * Пробелы заменяются на "-" или "_".

2. Вариант 2 — ввод строки с использованием calloc + realloc + free(cstring.cpp)
* Изначально выделяется буфер фиксированного размера (например, 5 байт).
* При переполнении буфер увеличивается через realloc на дополнительный блок.
* realloc автоматически копирует старые данные и при необходимости выделяет новый блок.
* После завершения ввода пробелы заменяются по правилу задачи.
  
3. Вариант 3 — работа с файлами(filestring.cpp)
* Строка считывается из файла input.txt посимвольно.
* Алгоритм выделения памяти повторяет логику первого варианта:
  каждое добавление символа вызывает перевыделение и копирование.
* После проверки строки пробелы заменяются на нужный символ.
* Результат записывается в output.txt.

Тестирование:
В проекте используется библиотека Catch2 для модульного тестирования.

* Все тесты находятся в файле test.cpp
* Тестируются:
  * Строки с цифрами и без,
  * Строки с множественными пробелами,
  * Граничные случаи (пустые строки, только пробелы),
  * Поведение при различных длинах,
  * Корректность замены,
  * Неизменность исходных данных,
  * Работа с особыми символами (\t, \n, спец. ASCII).
